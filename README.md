# Center of Gravity Finder is a Python project that analyzes synthetic transaction data to identify potentially suspicious accounts using network analysis, risk scoring, network relationships, temporal analysis, and geographic indicators.

## Features

- Generates Synthetic financial transaction data
- Scores accounts based on transaction behavior
- Detects multi-hop transaction networks
- Performs geographic risk analysis
- Exports results to CSV files

## Files

- generator.py - Generates synthetic transaction data
- analyzer.py - Analyzes transactions and calculates risk scores
- geo_analyzer.py - Performs geographic risk analysis
- cog_engine.py - Runs the project 
- transactions.csv - Sample transaction data

## Detection Methodology

This project analyzed synthetic financial transactions using a layered approach inspired by publicly available anti-money laundering (AML) guidance. COG-Finder identifies potentially suspicousaccounts by combining several behavior and geographic indicators into a single Center of Gravity (COG) score.

### Transaction Activity
The program calculates total transactions sent and received, transaction frequency, aggregate transaction values for each account.

### Multihop Network Analysis
Accounts are connected into a transaction graph . The analyzer identifies two-way activity as well as 3-hop, 4-hop, and 5-hop transaction chains. Longer transaction paths receive additional risk points because they may indicate attempts to obscure the movement of funds.

### Time-Window Analysis
Transactions are evaluated accross multiple time windows (7, 30, 60, 90 and 120 days). Accounts exceeding configure transaction thresholds within these windows receive additional risk points.

### Georgaphic Risk Analysis
Transactions are compared against selected high-riskgeographic locations. Accounts conducting transactions near these locations receive geographic risk scores based on proximity.

### Cent of Gravity (COG) Scoring
The final COG score combines multiple indicators, including:
- Transaction totals
- Number of connected accounts
- Multi-hop transaction behavior
- Time-Window flags
- Merchant and location risk indicators
- Two-way sender/receiver activity

Accounts are ranked by their total COG score, and summary CSV reports are generated for futher analysis. The scoring model is configurable, allowing investigators to adjust thresholds and weighing factors for different financial investigation scenarios.

## Data Sources and Inspiration

The synthetic data model and detection logic were inspired by publicly available guidance and reporting including:

- FinCEN advisories and trend analysis
- Bank Secrecy Act (BSA) reporting Guidance
- Public reports from the DHS, DEA, FBI, and other U.S. Government Agencies
- Open source reporting on money laudering, sanctions evasion, and illicit financial networks

No real financial records or personally identifiable information are included. All transaction data used by this project is sythetically generated for educational and research purposes.
